# ぽえとれ ローカル試作

## 今回の境界

- `src/poetore` はPoENavi本体の状態に依存しない独立モジュール。
- 公開APIは `parse_item_text()` と、その戻り値 `ParsedItem` / `ItemModifier`。
- UIだけがPySide6へ依存し、パーサーとモデルは標準ライブラリのみで動く。
- PoENavi本体は起動時に `poetore` をimportしない。💰ボタンを押した時だけ
  `src.poetore.ui` を遅延importして別ウィンドウを生成する。
- Trade APIへの実通信、ホットキー、検索結果UI、日本語Modマスタは次段階。未確認の
  API仕様やデータを今回の正本には入れていない。

## Windows実機確認後の改善

- 武器性能、装備条件、ソケットなど、アイテムレベル以前の区画をModから除外。
- 詳細コピーに含まれるPrefix / Suffix / Crafted等の見出しは件数へ含めず、次のModへ
  種別として付与する。
- 解析結果は項目列を内容幅、結果列を残り幅にし、解析後は必ず先頭から表示する。

## Awakened PoE Tradeから確認したこと

- 参照先: <https://github.com/SnosMe/awakened-poe-trade>
- 2026-07-19確認時点でMIT License（Copyright 2020 Alexander Drozdov）。
- 上流もクリップボード文を `--------` ごとのセクションへ分割し、名前、基礎データ、
  プロパティ、Modを段階的に解析している。
- 今回は設計境界だけを参考にしたクリーンなPython実装で、上流ソースやアイテムデータを
  コピー・改変していない。そのため現段階で第三者コードを配布物へ含めてはいない。
- 将来、上流のコードまたはデータを取り込む場合は、MIT本文と著作権表示を配布物へ含める。
