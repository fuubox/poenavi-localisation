# Upstream v2.6.2 Localization Merge Design

## Goal

Merge `buri34/poenavi` v2.6.2 (`b468cbe`) into the localized `main`
branch, preserve all upstream behavior, and ship complete Japanese and English
UI for the cheat-sheet overlay, hotkey changes, and Poetrieve fixes.

## Scope

The upstream range contains nine commits after v2.6.1. It adds a cheat-sheet
image overlay, centers overlays on the game monitor, improves empty-state
guidance, adds a configurable application-exit hotkey, clarifies the TCP logout
label, preserves corrupted Unique implicits, displays opalescent-oil
anointments, and bumps the application to v2.6.2.

The merge will:

- retain the complete upstream cheat-sheet overlay and positioning behavior;
- localize application-owned cheat-sheet labels, tooltips, empty-state text,
  dialog text, settings controls, and hotkey descriptions;
- preserve cheat-sheet filenames, image paths, JSON keys, hotkey action IDs,
  and persisted user content as operational data;
- retain the upstream exit-hotkey behavior without overwriting an existing
  user binding during configuration upgrade;
- clarify the TCP logout label in both locales without implying that PoENavi
  establishes a game connection;
- retain upstream Poetrieve corrupted-implicit and opalescent-oil behavior;
- preserve `Poetrieve` branding, parser and Trade API tokens, PoE1-only
  visibility/dispatch guards, and the existing `Alt+D` collision safeguard;
- preserve English guides, localization validation, release packaging, and
  user data;
- merge rather than rebase so upstream ancestry remains explicit.

## Integration Approach

Use a no-fast-forward, no-commit merge of `upstream/main` at `b468cbe` into the
clean local `main`. Resolve overlapping UI and configuration files by retaining
both upstream v2.6.2 behavior and the localization fork's display boundary.

New application-owned Japanese UI strings will be routed through `tr_ui` and
added as exact keys to `data/i18n/ui_ja.json` and `data/i18n/ui_en.json`.
Operational values such as action IDs, paths, filenames, PoE item-copy text,
Trade API identifiers, URLs, league IDs, and parser metadata will remain
unchanged.

## Configuration Compatibility

Existing schema-4 configurations will receive new defaults through the current
deep-merge mechanism. Before accepting a default exit shortcut, migration and
registration behavior will be checked for collisions with existing actions.
An explicit user value remains authoritative. New installations receive
upstream's intended exit shortcut.

The existing Poetrieve `Alt+D` collision safeguard and PoE1-only availability
checks must remain intact after conflict resolution.

## Error Handling and UI Behavior

Missing or empty cheat-sheet image sets will show localized guidance rather
than an invisible or unusable overlay. Image loading, monitor selection,
window placement, and persisted paths remain upstream behavior.

TCP logout wording will describe the configured action accurately: it attempts
to disconnect matching Path of Exile TCP connections; the application itself
does not maintain a gameplay connection.

If an English Path of Exile term is unclear after checking upstream aliases,
tests, and existing catalogs, implementation pauses for user guidance instead
of guessing.

## Verification

Verification will include:

- a clean pre-merge locale validator and full pytest baseline;
- locale-validator coverage for the new cheat-sheet module;
- English/Japanese construction checks for cheat-sheet and settings UI;
- focused cheat-sheet, configuration, hotkey, Poetrieve Trade, and corrupted
  implicit tests;
- explicit checks that existing hotkey bindings and PoE1-only Poetrieve
  protections survive;
- complete locale validation and pytest after independent review;
- confirmation that `b468cbe` is the second parent/ancestor of the merge
  commit and no localization resource was deleted.

After successful verification, create and push a merge commit on `main`, then
create and push annotated tag `v2.6.2` on that localized merge commit to
trigger the fork's release workflow.
