# Upstream v2.6.1 Poetrieve Localization Merge Design

## Goal

Merge `buri34/poenavi` v2.6.1 (`5a643a5`) into the localized `main`
branch, preserve upstream behavior, and keep both Japanese and English UI
complete under the approved English brand **Poetrieve**.

## Scope

The upstream range contains 22 commits after v2.6.0. It adds unique-item roll
range controls, currency icons and Divine/Chaos conversion display, logbook
layout changes, search-refresh behavior, documentation, tests, and the v2.6.1
version bump.

The merge will:

- preserve the upstream parser, Trade API, poe.ninja, metadata, and internal
  identifiers as operational data;
- route new application-owned Japanese display strings through `tr_ui`;
- add exact Japanese and English catalog entries using official Path of Exile
  terminology;
- retain the English brand `Poetrieve`;
- retain the localization fork's PoE1-only visibility and dispatch guards;
- retain the upgrade safeguard that prevents the new `Alt+D` default from
  stealing an existing hotkey;
- preserve all English guides, localization validators, packaging resources,
  and existing user data;
- merge rather than rebase so upstream ancestry remains explicit.

## Integration Approach

Use a no-fast-forward, no-commit merge of `upstream/main` into the clean local
`main`. Resolve conflicts by preserving both upstream v2.6.1 behavior and the
localized display boundary. Do not translate item-copy text, Trade API values,
currency identifiers, league IDs, URLs, parser fixtures, or metadata keys.

New currency artwork remains an upstream asset. Visible currency names,
tooltips, status messages, unique-roll labels, logbook labels, and conversion
text follow the active PoENavi locale.

## Compatibility and Error Handling

Existing configurations continue to load under schema 4. Poetrieve remains
disabled outside PoE1, and an existing `Alt+D` assignment remains authoritative
when upgrading. Network and parse errors continue to originate in the domain
layer and are localized only where displayed.

If an English rendering is unclear after checking the upstream English aliases,
Trade API identifiers, and official Path of Exile terminology, implementation
pauses for user guidance instead of guessing.

## Verification

Verification will include:

- a clean pre-merge locale validator and full pytest baseline;
- locale-validator coverage for every new Poetrieve UI literal;
- focused English/Japanese Poetrieve UI, Trade, parser, poe.ninja, hotkey, and
  distribution tests;
- explicit checks that the PoE1-only and hotkey-collision safeguards survive;
- the complete pytest suite and locale validator after review;
- confirmation that `5a643a5` is the second parent/ancestor of the merge
  commit and no localization resource was deleted.

After review and a successful merge commit, push `main` to `origin` as
explicitly approved by the user.
