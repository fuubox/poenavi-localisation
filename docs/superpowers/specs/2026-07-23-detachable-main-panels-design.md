# 切り離し可能なメインパネル設計

## 目的

タイマー、ガイド、マップを本体ウィンドウから個別に切り離し、必要なパネルだけを表示できるようにする。

## 対象と非対象

- 対象: タイマー、ガイド、マップ。
- 非対象: ゾーン情報、ジェム取得、みになび、既存の本体ウィンドウ操作。
- 切り離したパネルは本体から完全に取り除き、残ったパネルが縦方向に詰まる。

## UIと操作

- 対象パネルの見出しに `↗ 切り離す` ボタンを置く。
- 切り離し時は既存パネルのウィジェットを独立したフレームレスでないツールウィンドウへ移す。
- 独立ウィンドウには `↙ 本体へ戻す` ボタンを常時表示する。
- 復帰時はパネルを元の本体レイアウト位置へ戻す。
- 同じパネルを複数の独立ウィンドウに複製しない。各パネルの表示先は本体または独立ウィンドウのいずれか一方だけとする。

## 状態保存

設定の `detached_panels` にパネルIDごとの状態を保存する。

```json
{
  "timer": {"detached": true, "x": 100, "y": 120, "width": 420, "height": 280},
  "guide": {"detached": false},
  "map": {"detached": true, "x": 540, "y": 120, "width": 620, "height": 420}
}
```

- パネルIDは `timer`、`guide`、`map` の固定値とする。
- 切り離し・移動・リサイズ・復帰・終了時に状態を保存する。
- 起動時に `detached` が真なら、保存済みの位置とサイズで切り離して復元する。
- 保存済みの位置またはサイズが不正な場合は、画面内の既定サイズで表示する。

## 実装境界

- `src/ui/main_window.py` に、既存パネルの本体コンテナと元のレイアウト位置を登録する管理層を追加する。
- 独立ウィンドウは、パネル本体と復帰ボタンだけを所有する小さな専用ウィジェットにする。
- タイマー、ガイド、マップの内容更新ロジックは既存ウィジェットをそのまま使うため、更新経路を複製しない。

## テスト

- パネルを切り離すと本体レイアウトから外れ、独立ウィンドウに入ること。
- 本体へ戻すと元のレイアウト順に復帰すること。
- 切り離し状態とジオメトリを設定へ保存・復元できること。
- `QT_QPA_PLATFORM=minimal python -X faulthandler -m pytest -p no:unraisableexception -q` を実行すること。
