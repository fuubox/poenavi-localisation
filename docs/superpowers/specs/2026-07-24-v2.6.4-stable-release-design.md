# v2.6.4 Stable Release Design

## Goal

Publish v2.6.4 as the latest stable fork release so an existing v2.6.2
installation offers it even after the user dismissed v2.6.3. The package
includes the Markdown changelog renderer and curated release-note workflow
already verified on `main`.

## Version and Release Channel

Change `APP_VERSION` from `2.6.3` to `2.6.4`.

Create annotated tag `v2.6.4` on the verified release commit and push it
without force. Keep the release workflow in its normal stable configuration:
do not add `--prerelease`.

The existing v2.6.3 tag and prerelease remain unchanged.

## Bilingual Release Notes

Add `docs/releases/v2.6.4.md`, as required by the release workflow, with this
exact body:

```markdown
# English

## What's changed

- Update changelogs now render Markdown formatting and open links in the default browser.
- Releases now require curated bilingual notes instead of relying on sparse automatically generated notes.
- Added a tooltip explaining the Automatic / First time / Second time guide toggle.
- Fixed Japanese acquisition badges appearing beside gems in the English interface. They now display Quest or Buy.

# 日本語

## 変更内容

- アップデートの変更履歴がMarkdown形式で表示され、リンクを既定のブラウザで開けるようになりました。
- 内容の少ない自動生成リリースノートに依存せず、英語と日本語のリリースノートを必須にしました。
- ガイドの「自動 / 1回目 / 2回目」切り替えの動作を説明するツールチップを追加しました。
- 英語インターフェースのジェム入手情報に日本語のバッジが表示される問題を修正しました。英語では Quest または Buy と表示されます。

Full changelog / 完全な変更履歴:
https://github.com/fuubox/poenavi-localisation/compare/v2.6.2...v2.6.4
```

The comparison starts at v2.6.2 because v2.6.3 remains a prerelease and
v2.6.2 is the current stable update source.

## Build and Publication

Before tagging, run:

- the full pytest suite with the offscreen Qt platform;
- locale validation;
- `git diff --check`; and
- a release-contract check confirming `APP_VERSION == "2.6.4"`, the matching
  notes file exists, and the workflow has no `--prerelease` flag.

Push `main`, create annotated tag `v2.6.4`, and push the tag. Monitor both CI
and `Release Windows App` for the tag. The release workflow must:

1. match the tag to `APP_VERSION`;
2. find `docs/releases/v2.6.4.md`;
3. pass the full suite;
4. build `PoENavi.zip` and `PoENavi.zip.sha256`; and
5. publish a non-draft, non-prerelease GitHub release.

After publication, verify the release is the latest stable release, its body
matches `docs/releases/v2.6.4.md`, and both assets are uploaded.

## User Test

The user launches the existing packaged v2.6.2 application normally:

```powershell
.\PoENavi.exe
```

No `POENAVI_UPDATE_TEST_TAG` variable is required. The stored dismissed
version is v2.6.3, so it does not suppress the newer v2.6.4 prompt.

The first v2.6.4 prompt is rendered by the installed v2.6.2 code and therefore
shows Markdown as readable plain text. After the update installs v2.6.4,
future update prompts use formatted Markdown with clickable external links.

The test verifies prompt display, download, checksum, updater handoff,
restart, reported v2.6.4 version, and preservation of user data.
