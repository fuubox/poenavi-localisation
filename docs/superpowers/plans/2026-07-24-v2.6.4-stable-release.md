# v2.6.4 Stable Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish v2.6.4 as the latest stable fork release with curated bilingual notes and the updated Markdown-rendering updater dialog.

**Architecture:** Make the release contract derive its required notes file from `APP_VERSION`, then bump the version and add the matching bilingual notes. Verify locally, push `main`, create an immutable annotated tag, and let the existing GitHub Actions workflow test, package, and publish the stable release.

**Tech Stack:** Python 3.12, pytest/pytest-qt, PySide6, PowerShell, GitHub Actions, GitHub Releases.

## Global Constraints

- The application version is exactly `2.6.4` and the tag is exactly `v2.6.4`.
- Publish v2.6.4 as stable, not as a prerelease.
- Do not change or move the existing v2.6.3 tag or prerelease.
- Release notes are bilingual with English first, Japanese second, and compare v2.6.2 through v2.6.4.
- The release must retain both `PoENavi.zip` and `PoENavi.zip.sha256`.
- Never force-push or force-move the v2.6.4 tag.
